# GroupSync Deployment Guide - Render

## Prerequisites

- GitHub account
- Render account (free tier available)
- Code pushed to GitHub repository

---

## Part 1: Prepare for Deployment

### 1.1 Environment Variables

The following environment variables need to be configured in Render:

| Variable | Value | Description |
|----------|-------|-------------|
| `DJANGO_SECRET_KEY` | (auto-generated by Render) | Django secret key for production |
| `DEBUG` | `False` | Disable debug mode in production |
| `ALLOWED_HOSTS` | `your-app.onrender.com` | Your Render app URL |
| `CORS_ALLOWED_ORIGINS` | `https://your-frontend-url.com` | Frontend URL for CORS |
| `CSRF_TRUSTED_ORIGINS` | `https://your-app.onrender.com` | Trusted origins for CSRF |
| `PYTHON_VERSION` | `3.11.0` | Python version to use |

### 1.2 Files Checklist

Ensure these files exist in your repository:

- ✅ `requirements.txt` - Python dependencies
- ✅ `build.sh` - Build script for Render
- ✅ `render.yaml` - Render configuration (optional)
- ✅ `.env.example` - Environment variable template
- ✅ `.gitignore` - Includes `.env` to prevent committing secrets

## Part 2: Deploy Backend to Render

### Step 1: Create New Web Service

1. Log in to [Render Dashboard](https://dashboard.render.com/)
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub repository
4. Select the repository containing your Django backend

### Step 2: Configure Web Service

**Basic Settings:**
- **Name:** `groupsync-backend` (or your preferred name)
- **Environment:** `Python 3`
- **Region:** Choose closest to you
- **Branch:** `main`
- **Root Directory:** Leave blank (or specify if backend is in subdirectory)

**Build & Deploy:**
- **Build Command:** `./build.sh`
- **Start Command:** `gunicorn groupsync.wsgi:application`

### Step 3: Configure Environment Variables

In the Render dashboard, go to **Environment** tab and add:

```
DJANGO_SECRET_KEY=<click "Generate" for auto-generated value>
DEBUG=False
PYTHON_VERSION=3.11.0
```

**Important:** After deployment, add these with your actual URLs:

```
ALLOWED_HOSTS=your-app-name.onrender.com
CORS_ALLOWED_ORIGINS=https://your-frontend-url.com,http://localhost:3000
CSRF_TRUSTED_ORIGINS=https://your-app-name.onrender.com
```

### Step 4: Deploy

1. Click **"Create Web Service"**
2. Render will automatically:
   - Clone your repository
   - Run `build.sh` (install dependencies, collect static files, run migrations)
   - Start the application with gunicorn
3. Wait for deployment to complete (5-10 minutes)

### Step 5: Verify Backend Deployment

1. Once deployed, you'll get a URL like: `https://groupsync-backend.onrender.com`
2. Test endpoints:
   - `https://your-app.onrender.com/api/users/register/` (should return 405 Method Not Allowed for GET)
   - `https://your-app.onrender.com/admin/` (should show Django admin login)

## Part 3: Update Frontend Configuration

### Update Frontend Environment Variables

In your React app, update `.env`:

```
REACT_APP_API_URL=https://your-app-name.onrender.com
```

### Deploy Frontend (Optional - Render Static Site)

1. In Render Dashboard, click **"New +"** → **"Static Site"**
2. Connect same GitHub repository
3. Configure:
   - **Name:** `groupsync-frontend`
   - **Root Directory:** `groupsync-frontend`
   - **Build Command:** `npm install && npm run build`
   - **Publish Directory:** `build`
4. Add environment variable:
   ```
   REACT_APP_API_URL=https://your-backend-url.onrender.com
   ```

## Part 4: Post-Deployment

### 4.1 Update Backend CORS Settings

Once frontend is deployed, update backend environment variables in Render:

```
CORS_ALLOWED_ORIGINS=https://your-frontend.onrender.com
CSRF_TRUSTED_ORIGINS=https://your-backend.onrender.com,https://your-frontend.onrender.com
ALLOWED_HOSTS=your-backend.onrender.com
```

**Important:** After updating environment variables, Render will automatically redeploy.

### 4.2 Test Complete Flow

1. Navigate to your frontend URL
2. Test registration:
   - Create a new account
   - Verify email validation
   - Check that JWT tokens are returned
3. Test login:
   - Login with created account
   - Verify redirect to home/dashboard
4. Test protected routes:
   - Access profile page
   - Verify authentication is working
5. Test logout:
   - Logout
   - Verify redirect to login
   - Verify protected routes are blocked

### 4.3 Check Logs

If anything fails:

1. Go to Render Dashboard → Your Service
2. Click **"Logs"** tab
3. Check for errors in:
   - Build logs (deployment errors)
   - Server logs (runtime errors)

## Part 5: Common Issues & Solutions

### Issue: Build Fails

**Error:** `Could not find a version that satisfies the requirement`

**Solution:**
- Check `requirements.txt` for typos
- Ensure all packages are available for Python 3.11
- Pin specific versions

### Issue: Static Files Not Loading

**Error:** CSS/Admin panel looks broken

**Solution:**
- Verify `STATIC_ROOT` is set in `settings.py`
- Check `build.sh` runs `collectstatic`
- Ensure `whitenoise` is in `MIDDLEWARE` (should be after `SecurityMiddleware`)

### Issue: CORS Errors in Frontend

**Error:** `No 'Access-Control-Allow-Origin' header`

**Solution:**
- Update `CORS_ALLOWED_ORIGINS` with correct frontend URL
- Ensure no trailing slashes
- Check `CORS_ALLOW_CREDENTIALS = True` in settings.py
- Redeploy after changes

### Issue: Database Migrations Not Applied

**Error:** `no such table` or migration errors

**Solution:**
- Check that `build.sh` includes `python manage.py migrate`
- Manually run migrations in Render Shell:
  1. Go to service → Shell tab
  2. Run: `python manage.py migrate`

### Issue: 500 Internal Server Error

**Solution:**
- Check Render logs for specific error
- Common causes:
  - Missing environment variables
  - Database migration issues
  - Import errors
- Enable temporary debugging:
  1. Set `DEBUG=True` in Render env vars
  2. Check error page for details
  3. **Don't forget to set back to `False`!**

## Part 6: Maintenance

### Update Dependencies

1. Update `requirements.txt` locally
2. Test locally
3. Push to GitHub
4. Render will auto-deploy

### Database Backups

**Important:** Render free tier with SQLite doesn't persist data across deployments!

**Solutions:**
1. Upgrade to paid tier with PostgreSQL
2. Use external database service
3. For now: Keep development database as source of truth

### Manual Deploy

If auto-deploy is disabled:

1. Go to Render Dashboard → Your Service
2. Click **"Manual Deploy"** → **"Deploy latest commit"**

---

## Part 7: Rollback

If deployment breaks:

1. Go to Render Dashboard → Your Service → **"Events"** tab
2. Find previous successful deployment
3. Click **"Rollback"** on that deployment

---

## Security Checklist

Before going live:

- ✅ `DEBUG=False` in production
- ✅ Strong `DJANGO_SECRET_KEY` (generated by Render)
- ✅ HTTPS only (Render provides free SSL)
- ✅ `.env` file in `.gitignore`
- ✅ No hardcoded secrets in code
- ✅ `ALLOWED_HOSTS` restricted to your domain
- ✅ CORS restricted to your frontend URL
- ✅ SQL injection protection (Django ORM handles this)
- ✅ XSS protection (Django templates handle this)

## Monitoring

### Check Service Health

1. Render Dashboard → Your Service
2. Check metrics:
   - Response time
   - Memory usage
   - CPU usage
3. Set up alerts:
   - Go to Settings → Notifications
   - Add email/Slack for downtime alerts

### View Logs in Real-Time

```bash
# In Render Shell tab
tail -f /var/log/application.log
```

Or use Render's built-in log viewer in dashboard.

## Cost Optimization

**Free Tier Limitations:**
- Service spins down after 15 minutes of inactivity
- First request after spin-down takes 30-60 seconds
- 750 hours/month free (enough for 1 service 24/7)

**To Prevent Spin-Down:**
1. Upgrade to paid tier ($7/month)
2. Or use a service like UptimeRobot to ping your app every 10 minutes
